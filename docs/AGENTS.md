## Conventions

- Always update this index when adding a new document to `docs/`.
- Prefer `docs/` for product-level material (requirements, roadmap, as-is snapshots). Prefer `specs/` for implementation-level material (data models, protocols, module specs).
- Cross-link freely between `docs/` and `specs/` rather than duplicating content.
